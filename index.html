Access only after declaration

It's not possible to access a const or let until after declaration. 


                            console.log(a) // undefined
                            console.log(b) // ‚ö†Ô∏è ReferenceError

                            var a = 1
                            let b = 2

let or const

A const is read-only. Reassigning a value is not permitted.


                            const a = 1
                            console.log(a) // 1

                            const a = 2 // ‚ö†Ô∏è TypeError

const does not mean immutable


                            const a = [1, 2, 3]
                            a.push(4)

                            console.log(a) // [1, 2, 3, 4]

If the value is complex (e.g. arrays, objects), their contents can be modified.

Arrow functions

Comparison


                            // function expression
                            const foo = function (x, y) {
                                return x + y
                            }

                            // arrow function
                            const foo = (x, y) => x + y

Arrow function declarations don't exist:


                            // function declaration
                            function foo(x, y) {
                                return x + y
                            }

                            // no arrow function declaration equivalent
Function declarations are like variable declarations (var a = 1) and are hoisted so they're available in their scope.

( ) are optional for a single argument


                            // no arguments
                            () => 42

                            // 1 argument with optional ( )
                            a => a + 1
                            (a) => a + 1

                            // more than 1 argument
                            (a, b) => a * b

{ } are optional for single expressions


                            // single expression
                            (a, b) => a * b

                            // multiple expressions
                            (a, b) => {
                                a * 2
                                b * 3
                                return a * b
                            }

Omiting the { } implicitly returns


                            const multiply = (a, b) => a * b

                            // calling the function returns its result:
                            const result = multiply(3, 4)
                            console.log(result) // 12

this binding


                                    function Thing() {
                                        this.counter = 0

                                        // regular function
                                        setInterval(function() {
                                            this.counter++
                                        }, 1000)
                                    }

                                    const thing = new Thing()

                                    // wait a few seconds
                                    console.log(thing.counter) // 0
                                

                                    function Thing() {
                                        this.counter = 0

                                        // arrow function
                                        setInterval(() => {
                                            this.counter++
                                        }, 1000)
                                    }

                                    const thing = new Thing()

                                    // wait a few seconds
                                    console.log(thing.counter) // 3

Arrow functions don't create a new this binding. The binding of this is not dynamic, but lexical.

this binding caveat


                                    // using regular functions
                                    const someObject = {
                                        aMethod: function() {
                                            this.anotherMethod()
                                        },

                                        anotherMethod: function() {
                                            console.log('w00p')
                                        }
                                    }

                                    someObject.aMethod()
                                    // 'w00p'
                                

                                    // using arrow functions
                                    const someObject = {
                                        aMethod: () => {
                                            this.anotherMethod()
                                        },

                                        anotherMethod: () => {
                                            console.log('w00p')
                                        }
                                    }

                                    someObject.aMethod()
                                    // ‚ö†Ô∏è TypeError

Here, this is lexically bound and points to the global scope.
